## 1. Context

- **Why**: During DR-001 design work, analysis of the actual sandbox landscape (nix flake jail-nix, Anthropic srt, harness-internal bwrap/sandbox-exec) revealed that sandbox orchestration is already handled at other layers. Building a sandbox adapter protocol in autobahn would duplicate what already exists, and worse — autobahn cannot reach the composition level that nix flakes operate at.
- **Source Document**: `brief.md` §5.5 "Sandbox adapter layer", §4.3 bullet "sandbox adapters"
- **Governing Delta**: DE-001

## 2. Related Artefacts

- **Design Revision**: DR-001 §11 (Sandbox Landscape position statement)
- **External References**: `~/flakes/pub/jailed-agents.nix`, Anthropic `sandbox-runtime`

## 3. Summary of Changes

**brief.md changes:**

- §4.3 "sandbox adapters" in the `autobahn` owns list → remove. Replace with note that sandbox is opaque to autobahn; the caller provides a sandboxed or unsandboxed executable.
- §5.5 "Sandbox adapter layer" subsystem → remove as a required subsystem. Replace with a position statement noting that sandbox concerns are handled by nix flakes, srt, or harness internals, and autobahn carries sandbox policy metadata for observation only.
- §6.2 "Sandbox adapters" interface expectations → remove as a required interface. Replace with note that `RuntimePolicy.sandbox_profile` carries metadata.
- §7.4 "Adapter flexibility vs core simplicity" → update to note sandbox adapter was evaluated and explicitly excluded.
- §8 package layout `adapters/sandbox/` → remove.

**What is NOT changing:**

- The brief's overall architecture stance is preserved
- Harness adapters and session backends remain as designed
- The brief's non-goal "bubblewrap is the only sandbox backend" is reinforced (not weakened) by this change — autobahn doesn't model any sandbox backend

## 4. Consequences

**Positive:**
- Eliminates a protocol layer that would duplicate existing sandbox solutions
- Reduces autobahn's scope to what it genuinely orchestrates
- Cleaner three-layer ownership model (spec-driver configures → nix/srt sandboxes → autobahn orchestrates)

**Negative:**
- If a future need arises for autobahn-level sandbox verification (e.g. "confirm sandboxed before production access"), it would need to be added as an observation concern, not an orchestration concern
- autobahn cannot enforce sandbox policy — it trusts the caller

## 6. Notes

- The brief was written before the sandbox landscape was fully analysed. This revision reflects findings from examining jailed-agents.nix and Anthropic's sandbox-runtime.
- If the sandbox landscape changes significantly (e.g. a portable Python sandbox API emerges), this revision can be revisited.
