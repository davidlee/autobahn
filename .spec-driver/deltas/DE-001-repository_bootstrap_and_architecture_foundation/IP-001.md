## 1. Summary

- **Delta**: DE-001 — Repository bootstrap and architecture foundation
- **Design Revision**: [DR-001](./DR-001.md)
- **Desired Outcome**: A working `autobahn` Python library with typed models, adapter protocols, artifact integration, and a proven session lifecycle vertical slice (spawn/observe/terminate via subprocess backend)

## 2. Context & Constraints

- **Current State**: Greenfield — no source code, broken pyproject.toml, spec-driver scaffolding only
- **Target State**: Importable library (`from autobahn.api import ...`) with passing tests, clean lint, and one end-to-end integration test proving the architecture
- **Dependencies**:
  - spec-driver workflow YAML schemas (available now, v1)
  - spec-driver DE-109 (review state machine) — review operations deferred until it lands
  - spec-driver DE-108 (structured CLI output) — needed for review CLI consumption
  - spec-driver DE-110 (fixture exports) — hand-maintained fixtures until it lands
- **Constraints**:
  - Library-first — no CLI, daemon, or network transport
  - Track spec-driver conventions: hatch, >=3.12, ruff, 2-space indent, pytest-asyncio
  - v1 API: spawn/observe/terminate only (resume/pause deferred per DEC-012)
  - Review APIs internal/experimental (DEC-014) until spec-driver contract lands

## 6. Testing & Verification Plan

### Unit tests (phases 2-4)
- Pydantic model validation against YAML fixtures (VA-001)
- Artifact loader/writer with fixture files (VA-002)
- Transition planning for each workflow state/role combination
- API functions with mock adapters (VA-003)
- Reconciliation drift detection with mock session state

### Integration tests (phase 5)
- Real subprocess backend: spawn `echo "done"`, observe completion, verify lifecycle (VA-004)
- Full API flow: load_context → transition_from_handoff → spawn → observe → reconcile

### Contract tests (phase 2)
- Hand-maintained YAML fixtures derived from spec-driver schemas (degraded posture until DE-110)
- Each fixture carries comment noting spec-driver commit it was derived from
- Validates autobahn's pydantic models parse spec-driver's data without error

### Not in scope
- Real tmux backend in CI
- Real agent session (Claude Code, pi-mono)
- Review operations (gated on DE-108/109)
