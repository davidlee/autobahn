## 1. Summary & Context

- **Predecessor**: [DE-001](../DE-001-repository_bootstrap_and_architecture_foundation/DE-001.md) — repository bootstrap
- **Implementation Plan**: [IP-002](./IP-002.md)
- **Design Revision**: [DR-002](./DR-002.md)
- **Change Drivers**: AUD-001 findings F-001, F-002, F-005, F-007, F-011

## 2. Motivation

- DE-001 proved the architecture with a vertical slice but deferred 6 findings
- The most consequential gaps: Supervisor design smell (F-001), incorrect SessionHandle metadata (F-002), duplicated terminal-states constant (F-005), missing schema/version validation (F-007), and no sessions.yaml write support (F-011)
- Without sessions.yaml writes, autobahn cannot persist session state — a prerequisite for any real orchestration use
- Without schema validation, autobahn silently accepts malformed or version-mismatched YAML — a stated DR-001 requirement

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Schema/version validation in artifact loader (F-007) — reject unknown schema or unsupported version
  - Refactor observe out of Supervisor (F-001) — clean the `__new__` bypass
  - Supervisor populates SessionHandle role/artifact_id after backend.create (F-002)
  - Extract shared terminal-states constant (F-005)
  - Sessions.yaml write support (F-011) — spawn persists handles, reconcile updates status
- **Operational Constraints**: Library-first, no new dependencies beyond what DE-001 established
- **Dependencies**: None — all work is internal to autobahn

## 4. Out of Scope

- Review operations (gated on spec-driver DE-108/DE-109)
- Generated fixture exports (gated on spec-driver DE-110)
- Hardcoded handoff path fix (F-004) — low impact, deferred
- Additional harness adapters (pi-mono, Codex)
- Sessions.yaml locking/lease semantics (OQ-005 still open)

## 5. Approach Overview

- **Key Changes**:
  - `artifacts/loader.py` — validate `schema` and `version` fields before model_validate
  - `runtime/supervisor.py` — remove observe method; Supervisor only does spawn/terminate
  - `api/functions.py` — observe_session becomes a free async generator using backend directly
  - `runtime/supervisor.py` — spawn patches SessionHandle with role/artifact_id from context/plan
  - `models/enums.py` or `runtime/_constants.py` — shared TERMINAL_WORKFLOW_STATES
  - `artifacts/writer.py` (new) — write/update sessions.yaml
  - `api/functions.py` — spawn_role_session writes session entry; reconcile updates status

## 6. Verification Strategy

- **Acceptance Criteria**:
  - Loader rejects YAML with wrong schema marker or unsupported version
  - observe_session works without Supervisor instantiation
  - SessionHandle from spawn carries correct role and artifact_id
  - spawn_role_session writes to sessions.yaml
  - reconcile updates session status in sessions.yaml
  - All existing tests still pass (no regressions)
  - `just check` passes

## 7. Risks & Mitigations

- **Risk**: Sessions.yaml write introduces file I/O in previously pure-read paths — _Likelihood_: low — _Impact_: medium — _Mitigation_: Write via atomic rename; keep reads and writes in separate modules
- **Risk**: Schema validation may be too strict for forward-compatible consumption — _Likelihood_: low — _Impact_: low — _Mitigation_: Validate schema marker prefix, allow patch versions within v1

- **Future Deltas**: Review operations (after DE-108/DE-109), additional adapters, CLI wrapper
