# DR-002 – Internal hardening and session persistence

## 1. Executive Summary

- **Delta**: [DE-002](./DE-002.md)
- **Status**: draft
- **Last Updated**: 2026-03-22
- **Synopsis**: Close 5 AUD-001 findings — add schema validation to the artifact loader, refactor Supervisor to remove the observe hack, fix SessionHandle metadata, extract shared constants, and implement sessions.yaml write support.
- **Review history**: One external adversarial pass integrated (see §Appendix A).

## 2. Problem & Constraints

**Current behaviour:**

1. Artifact loader accepts any YAML regardless of schema marker or version. DR-001 §9 explicitly requires rejection of unknown schemas. (F-007)
2. `observe_session` creates a half-initialized Supervisor via `__new__`, bypassing the constructor. (F-001)
3. `SubprocessBackend.create` returns SessionHandles with `role=Role.OTHER` and `artifact_id=""`. Supervisor doesn't fix this. (F-002)
4. Terminal workflow states are defined independently in `transition.py` and `reconcile.py`. (F-005)
5. autobahn never writes to sessions.yaml. Session state is lost between invocations. (F-011)

**Constraints:**

- No new external dependencies
- DR-001 coupling rules remain in force: adapters/ depends only on models/
- Single-writer assumption (DEC-011) — per-artifact, per-process. No within-process concurrent writes to the same sessions.yaml.
- OQ-005 (sessions.yaml ownership authority) still open — design for the simple case (autobahn is sole writer), document the assumption
- POSIX platform assumption — atomic rename requires same-filesystem temp file
- reconcile remains read-only per DR-001 §8 ("does not auto-repair; returns diagnosis") — session status persistence is a separate operation (DEC-025)

## 3. Architecture Intent

No structural changes to the module dependency graph. All changes are within existing modules or add new leaf modules that follow established coupling rules.

```
models/enums.py        ← add TERMINAL_WORKFLOW_STATES (leaf, no new deps)
artifacts/schema.py    ← NEW: SCHEMA_VERSIONS map (shared between loader and writer)
artifacts/loader.py    ← add _validate_schema_marker before model_validate; imports from schema.py
artifacts/writer.py    ← NEW: pure write function (depends on models/ + schema.py)
runtime/supervisor.py  ← remove observe, patch handle metadata in spawn
runtime/transition.py  ← import TERMINAL_WORKFLOW_STATES from enums
runtime/reconcile.py   ← import constant; add session_id to DriftItem; remains read-only
api/functions.py       ← inline observe loop; spawn writes; new persist_session_statuses (sync, report-driven)
```

**Design observation:** After removing observe, Supervisor exists only for spawn and terminate. Spawn combines `harness.launch_spec` + `backend.create` + handle patching. Terminate combines `backend.terminate` + `harness.parse_exit`. The class is thin but still justifies its existence as the coordination point between harness and backend — a future delta could inline it if it doesn't grow. Not actionable now.

## 4. Design Detail

### 4.1 Schema validation (F-007, DEC-020)

New file `artifacts/schema.py` — shared schema version map:

```python
"""Schema version registry for spec-driver workflow artifacts.

Shared by loader (validation) and writer (serialization).
Write-version policy: autobahn writes at the highest supported version
for a given schema.
"""

SCHEMA_VERSIONS: dict[str, frozenset[int]] = {
  "supekku.workflow.state": frozenset({1}),
  "supekku.workflow.handoff": frozenset({1}),
  "supekku.workflow.review-index": frozenset({1}),
  "supekku.workflow.review-findings": frozenset({1, 2}),
  "supekku.workflow.sessions": frozenset({1}),
}

def write_version(schema: str) -> int:
  """Return the version to use when writing a schema file."""
  return max(SCHEMA_VERSIONS[schema])
```

Add to `artifacts/loader.py`, importing from `schema.py`:

```python
from autobahn.artifacts.schema import SCHEMA_VERSIONS

def _validate_schema_marker(
  data: dict,
  expected_schema: str,
  path: Path,
) -> None:
  """Validate schema and version fields in YAML data.

  Raises ArtifactContractError if schema is missing, unknown,
  or version not in the supported set.
  """
  schema = data.get("schema")
  version = data.get("version")

  if schema is None:
    raise ArtifactContractError(
      f"Missing 'schema' field in {path}"
    )

  if schema != expected_schema:
    raise ArtifactContractError(
      f"Unknown schema '{schema}' in {path}, "
      f"expected '{expected_schema}'"
    )

  supported = SCHEMA_VERSIONS.get(expected_schema, frozenset())
  if version not in supported:
    raise ArtifactContractError(
      f"Unsupported version {version} in {path}, "
      f"expected one of {sorted(supported)}"
    )
```

**Callsites in `load_workflow_dir`:**

1. For state.yaml (required): called before `_validate(WorkflowStateFile, ...)`:
   ```python
   state_data = _read_yaml(state_path)
   _validate_schema_marker(state_data, "supekku.workflow.state", state_path)
   state = _validate(WorkflowStateFile, state_data, state_path)
   ```

2. For each optional file: `_OPTIONAL_FILES` gains a schema key:
   ```python
   _OPTIONAL_FILES: dict[str, tuple[str, type[BaseModel]]] = {
     "handoff.current.yaml": ("supekku.workflow.handoff", HandoffFile),
     "review-index.yaml": ("supekku.workflow.review-index", ReviewIndexFile),
     "review-findings.yaml": ("supekku.workflow.review-findings", ReviewFindingsFile),
     "sessions.yaml": ("supekku.workflow.sessions", SessionsFile),
   }
   ```
   Schema validation called before `_validate()` for each present optional file.

**Forward compatibility:** When DE-110 lands with exported schemas, this hardcoded map is replaced by schema metadata from spec-driver. The frozenset-per-schema approach handles the review-findings v1→v2 coexistence that DR-001 anticipated.

### 4.2 Observe refactor (F-001, DEC-021)

**Remove from `runtime/supervisor.py`:** The `observe` method.

**Replace in `api/functions.py`:**

```python
async def observe_session(
  handle: SessionHandle,
  *,
  backend: SessionBackend,
  poll_interval: float = 5.0,
) -> AsyncIterator[SessionMetadata]:
  """Yield session metadata until session ends."""
  while True:
    meta = await backend.get_metadata(handle)
    yield meta
    if not meta.alive:
      return
    await asyncio.sleep(poll_interval)
```

No Supervisor involvement. The function signature is unchanged — this is a pure implementation refactor.

### 4.3 SessionHandle metadata (F-002, DEC-023)

In `Supervisor.spawn`, after `backend.create` returns, patch and return the corrected handle:

```python
async def spawn(self, context, plan, policy) -> OperationResult[SessionHandle]:
  launch = self.harness.launch_spec(
    work_dir=policy.work_dir, plan=plan, policy=policy,
  )
  session_id = f"{context.artifact_id}-{plan.target_role}"

  try:
    handle = await self.backend.create(session_id, launch)
  except Exception as exc:
    return OperationResult(success=False, error=f"Failed to create session: {exc}")

  # Patch metadata — backend doesn't have orchestration context
  handle = handle.model_copy(update={
    "role": plan.target_role,
    "artifact_id": context.artifact_id,
  })
  return OperationResult(success=True, value=handle)
```

The patched handle is what gets returned in the OperationResult. The backend's original handle (with placeholder values) is discarded.

### 4.5 Sessions.yaml writer (F-011, DEC-022)

New file `autobahn/artifacts/writer.py`:

```python
from autobahn.artifacts.schema import write_version

_SESSIONS_SCHEMA = "supekku.workflow.sessions"

def write_sessions_file(path: Path, sessions: SessionsFile) -> None:
  """Write sessions.yaml atomically.

  Uses temp file + os.rename for crash safety.
  Platform assumption: POSIX (same-filesystem atomic rename).
  """
  data = {
    "schema": _SESSIONS_SCHEMA,
    "version": write_version(_SESSIONS_SCHEMA),
    "artifact": sessions.artifact.model_dump(mode="json"),
    "sessions": [
      s.model_dump(mode="json") for s in sessions.sessions
    ],
  }
  tmp = path.with_suffix(".tmp")
  tmp.write_text(yaml.dump(data, default_flow_style=False, sort_keys=False))
  tmp.rename(path)
```

**Coupling:** `artifacts/writer.py` depends on `models/` and `artifacts/schema.py` (same package). No cross-layer coupling. Version value comes from the shared registry, not hardcoded.

**Explicit override of DR-001 §9 write protocol:** DR-001 specifies a 6-step write protocol including lock acquisition (sessions.lock) and schema validation of output. DR-002 implements steps 4-5 only (temp write + atomic rename). Rationale:
- **Lock:** DEC-011 guarantees single-writer per artifact per process. No concurrent process can race. Within-process, spawn and persist_session_statuses are sequential (see §4.6, §4.8). Lock adds complexity with no safety benefit in v1.
- **Schema validation of output:** Redundant. The writer constructs data from typed pydantic models and serializes deterministically. The version is sourced from `_SCHEMA_VERSIONS`. A round-trip test (VA-005) via `load_workflow_dir` validates that written files pass the reader's schema validation, closing the loop without runtime double-validation.

When OQ-005 resolves and multi-writer is possible, locking wraps this function.

### 4.6 Spawn writes sessions.yaml (DEC-026)

In `api/functions.py`, `spawn_role_session` gains a persistence step:

```python
async def spawn_role_session(...) -> OperationResult[SessionHandle]:
  supervisor = Supervisor(harness, backend)
  result = await supervisor.spawn(context, plan, policy)
  if result.success and result.value:
    _persist_new_session(context, result.value, harness.name, policy)
  return result
```

**`_persist_new_session` is a local helper that:**
1. Reads existing sessions.yaml from `context.artifact_dir / "sessions.yaml"` (or creates empty SessionsFile with artifact block from context)
2. Appends a new SessionEntry from the handle
3. Writes via `write_sessions_file`

**Error handling (DEC-026):** If the write fails (I/O error), spawn_role_session:
- Still returns `OperationResult(success=True, value=handle)` — the session is running
- Appends the write error to `result.warnings`
- Does NOT terminate the session — persistence failure is recoverable (reconcile or a subsequent spawn can reconstruct state from the backend)

The session is the primary artifact; the sessions.yaml entry is bookkeeping. Failing to persist bookkeeping should not kill a successfully launched session.

**Sequential execution guarantee:** `spawn_role_session` is an async function, but the read-modify-write within `_persist_new_session` is synchronous I/O (Path.read_text / Path.write_text) — it does not yield to the event loop mid-cycle. Concurrent spawns for the same artifact would require the caller to `await` them sequentially or use a per-artifact lock. DEC-011 (single-writer per artifact) makes this the caller's responsibility. The design does not add internal locking.

### 4.7 Reconcile remains read-only (DEC-025)

**No changes to reconcile's write behaviour.** reconcile continues to return `OperationResult[ReconciliationReport]` with drift items. It does not write to sessions.yaml.

This preserves DR-001 §8's contract: "Does not auto-repair. Returns diagnosis."

### 4.8 New function: persist_session_statuses

New public API function in `api/functions.py`. **Sync, report-driven, no backend parameter.** Derives dead session IDs from the ReconciliationReport's drift items rather than re-probing liveness.

```python
def persist_session_statuses(
  context: WorkflowContext,
  report: ReconciliationReport,
) -> None:
  """Update sessions.yaml to reflect reconciliation findings.

  Caller invokes this after inspecting a ReconciliationReport.
  Marks sessions as dead based on drift items — does not re-probe
  liveness (reconcile already did that).

  Sync function: reads sessions.yaml, applies status changes, writes.
  """
```

**Prerequisite: add `session_id` to DriftItem.** The reconcile function already knows which session each drift item refers to. Adding an optional `session_id: str | None = None` field to `DriftItem` makes the data self-describing instead of requiring string parsing:

```python
class DriftItem(BaseModel):
  kind: str
  description: str
  session_id: str | None = None  # set for session-scoped drift
```

**Logic:**
1. Collect dead session IDs from report drift items where `kind` is `SESSION_DIED_UNEXPECTEDLY`, `SESSION_OUTLIVED_WORKFLOW`, or `ORPHANED_SESSION`
2. Read sessions.yaml from `context.artifact_dir` (return early if not present)
3. Build new entry list: for each entry, if `entry.session_id` is in the dead set and `entry.status == ACTIVE`, produce `entry.model_copy(update={"status": SessionStatus.DEAD})`. Otherwise keep unchanged.
4. If any entries changed, write via `write_sessions_file`

**This covers all failure cases:**
- Session died while workflow active → `SESSION_DIED_UNEXPECTEDLY` drift → mark dead
- Session outlived terminal workflow → `SESSION_OUTLIVED_WORKFLOW` drift → mark dead
- Orchestrator crash/restart, no handles → `ORPHANED_SESSION` drift → mark dead
- Active session with live handle → no drift item → unchanged

**Caller flow:**
```python
report = await reconcile(context, policy=policy, backend=backend, active_handles=handles)
if report.value and report.value.has_drift:
  # Inspect report, decide what to do
  persist_session_statuses(context, report.value)
```

The caller decides whether to persist. reconcile diagnoses; the caller prescribes.

## 7. Design Decisions

- **DEC-020**: Schema validation uses exact match on schema string; version checked against per-schema `frozenset[int]` of supported versions. Multi-version support for review-findings v1+v2.
- **DEC-021**: `observe` is a free async generator in `api/functions.py`. It only needs a backend.
- **DEC-022**: Sessions.yaml writer is a pure function with atomic rename. Explicit override of DR-001 §9 write protocol (no lock, no output schema validation) — justified by single-process guarantee and programmatic construction from typed models.
- **DEC-023**: Supervisor.spawn patches SessionHandle via `model_copy(update=...)` after backend.create. Returns the patched handle.
- **DEC-024**: `TERMINAL_WORKFLOW_STATES` is a `frozenset[WorkflowStatus]` in `models/enums.py`.
- **DEC-025**: reconcile remains read-only per DR-001 §8 ("diagnosis not prescription"). Session status persistence is `persist_session_statuses` — a separate sync, report-driven function. No backend parameter; derives dead sessions from ReconciliationReport drift items.
- **DEC-026**: Spawn persistence failure → `OperationResult` success with warning. Session is running regardless; persistence is recoverable bookkeeping.
- **DEC-027**: `SCHEMA_VERSIONS` map lives in `artifacts/schema.py` (public name, shared module). Write-version policy: autobahn writes at the highest supported version for a given schema.
- **DEC-028**: `DriftItem` gains `session_id: str | None` field so persist_session_statuses can derive dead sessions without string parsing.
