# DR-002 – Internal hardening and session persistence

## 1. Executive Summary

- **Delta**: [DE-002](./DE-002.md)
- **Status**: draft
- **Last Updated**: 2026-03-22
- **Synopsis**: Close 5 AUD-001 findings — add schema validation to the artifact loader, refactor Supervisor to remove the observe hack, fix SessionHandle metadata, extract shared constants, and implement sessions.yaml write support.

## 2. Problem & Constraints

**Current behaviour:**

1. Artifact loader accepts any YAML regardless of schema marker or version. DR-001 §9 explicitly requires rejection of unknown schemas. (F-007)
2. `observe_session` creates a half-initialized Supervisor via `__new__`, bypassing the constructor. (F-001)
3. `SubprocessBackend.create` returns SessionHandles with `role=Role.OTHER` and `artifact_id=""`. Supervisor doesn't fix this. (F-002)
4. Terminal workflow states are defined independently in `transition.py` and `reconcile.py`. (F-005)
5. autobahn never writes to sessions.yaml. Session state is lost between invocations. (F-011)

**Constraints:**

- No new external dependencies
- No public API signature changes (behavioural changes to spawn/reconcile are side-effect additions, not contract breaks)
- Single-writer assumption (DEC-011) — no locking in v1
- DR-001 coupling rules remain in force: adapters/ depends only on models/
- OQ-005 (sessions.yaml ownership authority) still open — design for the simple case (autobahn is sole writer), document the assumption

## 3. Architecture Intent

No structural changes to the module dependency graph. All changes are within existing modules or add new leaf modules that follow established coupling rules.

```
models/enums.py        ← add TERMINAL_WORKFLOW_STATES (leaf, no new deps)
models/artifacts.py    ← add schema/version fields to SessionsFile
artifacts/loader.py    ← add _validate_schema_marker before model_validate
artifacts/writer.py    ← NEW: pure write function (depends on models/ only)
runtime/supervisor.py  ← remove observe, patch handle metadata in spawn
runtime/transition.py  ← import TERMINAL_WORKFLOW_STATES from enums
runtime/reconcile.py   ← import constant; add sessions.yaml write after drift check
api/functions.py       ← inline observe loop; spawn/reconcile call writer
```

## 4. Design Detail

### 4.1 Schema validation (F-007, DEC-020)

Add to `artifacts/loader.py`:

```python
_SCHEMA_VERSIONS: dict[str, int] = {
  "supekku.workflow.state": 1,
  "supekku.workflow.handoff": 1,
  "supekku.workflow.review-index": 1,
  "supekku.workflow.review-findings": 2,
  "supekku.workflow.sessions": 1,
}

def _validate_schema_marker(
  data: dict,
  expected_schema: str,
  path: Path,
) -> None:
  """Validate schema and version fields in YAML data.

  Raises ArtifactContractError if schema is missing, unknown,
  or version doesn't match.
  """
  schema = data.get("schema")
  version = data.get("version")

  if schema is None:
    raise ArtifactContractError(
      f"Missing 'schema' field in {path}"
    )

  if schema != expected_schema:
    raise ArtifactContractError(
      f"Unknown schema '{schema}' in {path}, "
      f"expected '{expected_schema}'"
    )

  expected_version = _SCHEMA_VERSIONS[expected_schema]
  if version != expected_version:
    raise ArtifactContractError(
      f"Unsupported version {version} in {path}, "
      f"expected {expected_version}"
    )
```

Called in `load_workflow_dir` before each `_validate()` call. The `_OPTIONAL_FILES` dict gains a schema key:

```python
_OPTIONAL_FILES: dict[str, tuple[str, type[BaseModel]]] = {
  "handoff.current.yaml": ("supekku.workflow.handoff", HandoffFile),
  "review-index.yaml": ("supekku.workflow.review-index", ReviewIndexFile),
  "review-findings.yaml": ("supekku.workflow.review-findings", ReviewFindingsFile),
  "sessions.yaml": ("supekku.workflow.sessions", SessionsFile),
}
```

**Forward compatibility note:** When DE-110 lands with exported schemas, this hardcoded map gets replaced by schema metadata from spec-driver. The exact match strategy is correct for now — version bumps in spec-driver are intentional breaking changes that autobahn's contract tests should catch.

### 4.2 Observe refactor (F-001, DEC-021)

**Remove from `runtime/supervisor.py`:** The `observe` method.

**Replace in `api/functions.py`:**

```python
async def observe_session(
  handle: SessionHandle,
  *,
  backend: SessionBackend,
  poll_interval: float = 5.0,
) -> AsyncIterator[SessionMetadata]:
  """Yield session metadata until session ends."""
  while True:
    meta = await backend.get_metadata(handle)
    yield meta
    if not meta.alive:
      return
    await asyncio.sleep(poll_interval)
```

No Supervisor involvement. The function signature is unchanged — this is a pure implementation refactor.

### 4.3 SessionHandle metadata (F-002, DEC-023)

In `Supervisor.spawn`, after `backend.create` returns:

```python
handle = await self.backend.create(session_id, launch)
handle = handle.model_copy(update={
  "role": plan.target_role,
  "artifact_id": context.artifact_id,
})
```

`model_copy(update=...)` is pydantic's immutable update pattern. The backend creates the handle with placeholder values (it doesn't have orchestration context), and the Supervisor — which does have context — corrects them.

### 4.4 Terminal states constant (F-005, DEC-024)

Add to `autobahn/models/enums.py`:

```python
TERMINAL_WORKFLOW_STATES: frozenset[WorkflowStatus] = frozenset({
  WorkflowStatus.APPROVED,
})
```

Located in the leaf module (`models/enums.py`) so both `runtime/transition.py` and `runtime/reconcile.py` can import it without creating new coupling.

Delete the local `_TERMINAL_STATES` / `_TERMINAL_WORKFLOW_STATES` sets from both modules.

### 4.5 Sessions.yaml writer (F-011, DEC-022)

New file `autobahn/artifacts/writer.py`:

```python
def write_sessions_file(path: Path, sessions: SessionsFile) -> None:
  """Write sessions.yaml atomically.

  Uses temp file + rename for crash safety.
  """
  data = {
    "schema": "supekku.workflow.sessions",
    "version": 1,
    "artifact": sessions.artifact.model_dump(mode="json"),
    "sessions": [
      s.model_dump(mode="json") for s in sessions.sessions
    ],
  }
  tmp = path.with_suffix(".tmp")
  tmp.write_text(yaml.dump(data, default_flow_style=False, sort_keys=False))
  tmp.rename(path)
```

**Coupling:** `artifacts/writer.py` depends on `models/` only — same as `artifacts/loader.py`. No coupling rule changes.

**SessionsFile round-trip:** Add `schema_` and `version` fields to the model (or handle externally in the writer). Since `extra="ignore"` drops them on read, and the writer serializes them explicitly, keeping them out of the model is cleaner — the writer knows the authoritative schema/version. The reader's job is to validate them (§4.1), not store them.

### 4.6 Spawn writes sessions.yaml

In `api/functions.py`, `spawn_role_session` gains a write step:

```python
async def spawn_role_session(...) -> OperationResult[SessionHandle]:
  supervisor = Supervisor(harness, backend)
  result = await supervisor.spawn(context, plan, policy)
  if result.success and result.value:
    _persist_session_entry(context, result.value, harness.name, policy)
  return result
```

Where `_persist_session_entry` is a local helper that:
1. Reads existing sessions.yaml (or creates empty SessionsFile)
2. Appends a new SessionEntry from the handle
3. Writes via `write_sessions_file`

### 4.7 Reconcile writes sessions.yaml

In `runtime/reconcile.py`, after checking handle liveness, dead sessions get their status updated in sessions.yaml:

```python
# After drift detection, update sessions.yaml with current statuses
if fresh_context.sessions:
  updated = False
  for entry in fresh_context.sessions.sessions:
    if entry.status == SessionStatus.ACTIVE:
      handle_ids = {h.session_id for h in (active_handles or [])}
      if entry.session_id in handle_ids:
        # Check if actually dead
        for h in active_handles:
          if h.session_id == entry.session_id:
            alive = await backend.is_alive(h)
            if not alive:
              entry.status = SessionStatus.DEAD
              updated = True
  if updated:
    write_sessions_file(
      context.artifact_dir / "sessions.yaml",
      fresh_context.sessions,
    )
```

**Behavioural change:** `reconcile` was previously read-only. It now writes to sessions.yaml as a side effect when it detects dead sessions. This is documented in the API function's docstring but doesn't change the function signature.

## 7. Design Decisions

- **DEC-020**: Schema validation uses exact match on schema string and version integer, with a per-schema version map. When DE-110 lands, the hardcoded map is replaced by exported metadata.
- **DEC-021**: `observe` is a free async generator in `api/functions.py`. It only needs a backend; bundling it in Supervisor was a leaky coupling.
- **DEC-022**: Sessions.yaml writer is a pure function with atomic temp+rename. No locking in v1 (DEC-011 single-writer). When OQ-005 resolves, locking may be added as a wrapper.
- **DEC-023**: Supervisor.spawn patches SessionHandle metadata via `model_copy(update=...)` after backend.create. The backend doesn't have orchestration context; the Supervisor does.
- **DEC-024**: `TERMINAL_WORKFLOW_STATES` is a `frozenset[WorkflowStatus]` in `models/enums.py`. Leaf module, importable by any layer without new coupling.

## 8. Open Questions

None. All questions from AUD-001 findings are resolved by the design above. OQ-005 (sessions.yaml ownership authority) remains open at the cross-repo level but doesn't block v1 single-writer semantics.
