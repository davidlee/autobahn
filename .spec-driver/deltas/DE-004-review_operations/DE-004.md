## 1. Summary & Context

- **Predecessor**: DE-003 (schema reconciliation), DE-002 (orchestration patterns)
- **Implementation Plan**: [IP-004](./IP-004.md)
- **Design Revision**: [DR-004](./DR-004.md)
- **Change Drivers**: spec-driver DE-108/DE-109 provide the review CLI contract; ADR-002 establishes Python public API as the orchestrator integration boundary
- **Architectural Decision**: ADR-002 — CLI for agents, Python imports for orchestrators

## 2. Motivation

- DR-001 defined `prime_review` and `verify_review_outcome` as internal/experimental APIs, gated on spec-driver's review CLI contract (OQ-001)
- OQ-001 is now answered: DE-108 provides `--format json` with versioned envelope; DE-109 provides the review state machine, judgment lifecycle, and finding disposition commands
- ADR-002 redirects the integration mechanism: autobahn imports `spec_driver.workflow` directly instead of invoking CLI subprocess — simpler, typed, no serialization round-trip
- Without review operations, autobahn cannot orchestrate the reviewer role — it can spawn a reviewer session but cannot bootstrap review context or record outcomes

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `prime_review(delta_id, *, root)` — orchestrate review priming via `spec_driver.workflow` imports (evaluate staleness, build review index, write files, transition judgment state)
  - `summarize_review_outcome(context)` — read review findings from loaded context, return typed summary (round, status, finding counts, disposition completeness)
  - Finding disposition wrappers — resolve, defer, waive, supersede via `spec_driver.workflow` imports
  - `ReviewBootstrapState` and `ReviewOutcomeSummary` model types
  - Public API exports for review operations
- **Dependency**: spec-driver delta creating `spec_driver.workflow` public API surface (re-export façade)
- **Operational Constraints**: Library-first, no new external dependencies. Direct Python imports per ADR-002.

## 4. Out of Scope

- Review session orchestration (spawn reviewer, observe, terminate based on review outcome) — follow-up delta
- Automatic finding disposition (agent-driven waive/defer decisions) — policy question, not this delta
- Review round session metadata schema (autobahn owns per DEC-109-009; define shape when needed)
- Additional harness adapters (pi-mono, Codex)
- CLI tool adapter / subprocess invocation pattern (superseded by ADR-002)

## 5. Approach Overview

- **Key Changes**:
  - `models/runtime.py` — add `ReviewBootstrapState`, `ReviewOutcomeSummary`
  - `api/functions.py` — add `prime_review`, `summarize_review_outcome`, finding disposition functions
  - `api/__init__.py` — export new public API functions
- **Removed from original scope** (per ADR-002):
  - `adapters/tool/protocol.py` — no tool invocation protocol needed
  - `adapters/tool/spec_driver.py` — no CLI adapter needed
  - JSON envelope parsing — no envelope; direct Python calls
  - Exit code mapping — no exit codes; Python exceptions

## 6. Verification Strategy

- **Acceptance Criteria**:
  - `prime_review` orchestrates review priming and returns typed `ReviewBootstrapState`
  - `summarize_review_outcome` inspects loaded context and returns `ReviewOutcomeSummary`
  - Finding disposition functions call through to spec-driver domain logic and return structured result
  - Error handling: spec-driver exceptions propagated or mapped to autobahn error taxonomy
  - `uv run pytest` passes (both packages)
  - `uv run ruff check --fix autobahn tests && uv run ruff format autobahn tests` clean

## 7. Risks & Mitigations

- **Risk**: spec-driver public API surface not yet created — _Likelihood_: certain — _Impact_: blocking — _Mitigation_: Spec-driver delta is a prerequisite; can design autobahn side in parallel
- **Risk**: spec-driver internal refactors break imports — _Likelihood_: low — _Impact_: low — _Mitigation_: Re-export façade isolates autobahn from internal module paths
- **Risk**: Tight coupling between packages — _Likelihood_: inherent — _Impact_: medium — _Mitigation_: Narrow, auditable public surface (~30 symbols); contract tests catch drift
