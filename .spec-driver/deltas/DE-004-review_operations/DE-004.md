## 1. Summary & Context

- **Predecessor**: DE-003 (schema reconciliation), DE-002 (orchestration patterns)
- **Implementation Plan**: [IP-004](./IP-004.md)
- **Design Revision**: [DR-004](./DR-004.md)
- **Change Drivers**: spec-driver DE-108/DE-109 provide the review CLI contract; ADR-002 establishes Python public API as the orchestrator integration boundary; DE-124 creates the `spec_driver.orchestration` public surface
- **Architectural Decision**: ADR-002 — CLI for agents, Python imports for orchestrators

## 2. Motivation

- DR-001 defined `prime_review` and `verify_review_outcome` as internal/experimental APIs, gated on spec-driver's review CLI contract (OQ-001)
- OQ-001 is now answered: DE-108 provides `--format json` with versioned envelope; DE-109 provides the review state machine, judgment lifecycle, and finding disposition commands
- ADR-002 redirects the integration mechanism: autobahn imports `spec_driver.orchestration` directly instead of invoking CLI subprocess — simpler, typed, no serialization round-trip
- DE-124 extracts spec-driver's review orchestration into domain functions and exposes them through `spec_driver.orchestration` — autobahn calls composed operations, not building blocks
- Without review operations, autobahn cannot orchestrate the reviewer role — it can spawn a reviewer session but cannot bootstrap review context or record outcomes

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `prime_review(delta_id, *, root)` — thin wrapper around `spec_driver.orchestration.prime_review()`; returns `PrimeResult` (spec-driver type) or autobahn alias
  - `summarize_review_outcome(delta_dir)` — thin wrapper around `spec_driver.orchestration.summarize_review()`; returns spec-driver's `ReviewSummary`
  - `disposition_finding(delta_id, finding_id, *, action, root, ...)` — thin wrapper around `spec_driver.orchestration.disposition_finding()`
  - Review-related enums retired from autobahn — imported from `spec_driver.orchestration`
  - Public API exports for review operations
  - Remove dead `ToolInvocationError`/`ToolContractError` exception classes
- **Dependency**: spec-driver DE-124 (public API surface with composed operations)
- **Operational Constraints**: Library-first, no new external dependencies. Direct Python imports per ADR-002.

## 4. Out of Scope

- Review session orchestration (spawn reviewer, observe, terminate based on review outcome) — follow-up delta
- Automatic finding disposition (agent-driven waive/defer decisions) — policy question, not this delta
- Review round session metadata schema (autobahn owns per DEC-109-009; define shape when needed)
- Additional harness adapters (pi-mono, Codex)
- CLI tool adapter / subprocess invocation pattern (superseded by ADR-002)
- Disposition validation rules — owned by spec-driver (DR-109 domain logic, enforced in DE-124 operations)

## 5. Approach Overview

- **Key Changes**:
  - `models/enums.py` — retire review enums, re-export from `spec_driver.orchestration`
  - `models/artifacts.py` — update imports for review types
  - `models/errors.py` — remove `ToolInvocationError`, `ToolContractError`
  - `api/functions.py` — add `prime_review`, `summarize_review_outcome`, `disposition_finding` as thin wrappers
  - `api/__init__.py` — export new public API functions
- **Removed from original scope** (per ADR-002):
  - `adapters/tool/protocol.py` — no tool invocation protocol needed
  - `adapters/tool/spec_driver.py` — no CLI adapter needed
  - JSON envelope parsing — no envelope; direct Python calls
  - Exit code mapping — no exit codes; Python exceptions
- **Moved to spec-driver** (per cross-review F2/F3):
  - `ReviewBootstrapState` / `ReviewOutcomeSummary` — spec-driver's `PrimeResult` / `ReviewSummary` used directly or aliased
  - Disposition constraint validation — spec-driver domain logic, not autobahn orchestration policy

## 6. Verification Strategy

- **Acceptance Criteria**:
  - `prime_review` calls spec-driver's composed operation and returns typed result
  - `summarize_review_outcome` calls spec-driver's query operation and returns typed summary
  - `disposition_finding` calls spec-driver's composed operation
  - spec-driver exceptions propagate to callers (no mapping)
  - `uv run pytest` passes (both packages)
  - `uv run ruff check --fix autobahn tests && uv run ruff format autobahn tests` clean

## 7. Risks & Mitigations

- **Risk**: spec-driver DE-124 not yet complete — _Likelihood_: certain — _Impact_: blocking — _Mitigation_: DE-124 is in progress; design is cross-reviewed and aligned
- **Risk**: spec-driver internal refactors break imports — _Likelihood_: low — _Impact_: low — _Mitigation_: Re-export façade isolates autobahn from internal module paths
