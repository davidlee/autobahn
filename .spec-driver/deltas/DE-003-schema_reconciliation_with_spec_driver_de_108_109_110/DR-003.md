# DR-003 – Schema reconciliation with spec-driver DE-108/109/110

## 1. Executive Summary

- **Delta**: [DE-003](./DE-003.md)
- **Status**: draft
- **Last Updated**: 2026-03-22
- **Synopsis**: Reconcile autobahn's pydantic models with canonical spec-driver schemas after DE-108/109/110 landed. Enum removals, sessions.yaml structural change, review model enrichment.
- **Governing ADR**: ADR-001 — spec-driver schema authority and autobahn extension contract

## 2. Problem & Constraints

**Current behaviour:**

1. `BootstrapStatus` includes `WARMING` — removed upstream (DEC-109-002, prime is atomic).
2. `ReviewStatus` includes `BLOCKED` — removed upstream (DEC-109-003, parent workflow owns blocking).
3. `SessionsFile.sessions` is `list[SessionEntry]` — canonical schema is `dict[str, SessionEntry]` keyed by role. Writer produces non-conformant YAML.
4. `FindingDisposition.authority` is `str | None` — canonical type is `DispositionAuthority` enum. Missing 5 fields needed for disposition validation.
5. `Finding` is missing required `title` field; `summary` should be optional.
6. `ReviewRound` missing `reviewer_role`, `completed_at`, `summary`, `session` (autobahn extension point per DEC-109-009).
7. `ReviewBlock` missing `last_bootstrapped_at`, `source_handoff`; `scope` should be `session_scope`.
8. `StalenessKeyBlock` has `state_sha` (not canonical); missing `phase_id`.

**Constraints:**

- ADR-001: spec-driver is schema authority; autobahn conforms
- `extra="ignore"` on all models (DEC-001/DEC-015) — read a subset, must not contradict
- Writer must produce YAML that spec-driver's `MetadataValidator` accepts (ADR-001 §1)
- `MetadataValidator` checks required fields and types but does NOT reject extra fields — confirmed by probing spec-driver source
- Spec-driver never reads sessions.yaml — autobahn is sole reader and writer
- No new API functions — model + contract layer only
- No new dependencies

**Out of scope (DEC-032):**

- `domain_map`, `invariants`, `risk_areas`, `review_focus`, `known_decisions` on ReviewIndexFile
- Handoff `verification`, `git`, `design_tensions`, `unresolved_assumptions`, `decisions_to_preserve`
- Handoff `OpenItemBlock` restructure (id, blocking fields)
- These are deferred until autobahn needs them; `extra="ignore"` tolerates their presence
- **Architecture justification:** DE-004 review operations invoke spec-driver CLI commands (`review prime`, `review complete`). The reviewer agent reads workflow YAML files directly. autobahn does not need to relay domain_map or other review context through its models — it orchestrates the session, not the review content.

## 3. Architecture Intent

No structural changes to the module dependency graph. All changes are within existing modules. The sessions.yaml model change is the only one that affects write paths and runtime logic; everything else is model field additions/corrections that flow through existing `extra="ignore"` loading.

```
models/enums.py        ← remove WARMING, BLOCKED
models/artifacts.py    ← SessionEntry/SessionsFile restructure; Finding/Disposition/Round/ReviewBlock updates
artifacts/schema.py    ← no changes (schema versions unchanged)
artifacts/writer.py    ← produce role-keyed dict
artifacts/loader.py    ← no structural changes (pydantic handles shape)
api/functions.py       ← _persist_new_session, persist_session_statuses updated for dict
runtime/reconcile.py   ← orphan detection updated for dict-keyed sessions
tests/                 ← fixtures, contract tests, writer/loader/API/reconcile tests
```

## 4. Design Detail

### 4.1 Enum removals

**`models/enums.py`:**

```python
class BootstrapStatus(StrEnum):
  COLD = "cold"
  WARM = "warm"
  STALE = "stale"
  REUSABLE = "reusable"
  INVALID = "invalid"
  # WARMING removed (DEC-109-002: prime is atomic, no observable intermediate)

class ReviewStatus(StrEnum):
  NOT_STARTED = "not_started"
  IN_PROGRESS = "in_progress"
  CHANGES_REQUESTED = "changes_requested"
  APPROVED = "approved"
  # BLOCKED removed (DEC-109-003: parent workflow owns blocking)
```

Member order verified against canonical `review_state_machine.py`:
- `BootstrapStatus`: `COLD, WARM, STALE, REUSABLE, INVALID` — autobahn's current order after removing `WARMING` needs `REUSABLE` moved before `INVALID`.
- `ReviewStatus`: `NOT_STARTED, IN_PROGRESS, APPROVED, CHANGES_REQUESTED` — matches autobahn's current order (minus `BLOCKED`). No reordering needed.

Pre-removal: grep for `WARMING` and `BLOCKED` usage in autobahn source to confirm no runtime references beyond enum definition and contract test.

### 4.2 Sessions.yaml model restructure (DEC-030, DEC-031)

**Target model:**

```python
class SessionEntry(BaseModel):
  """Per-role session record in sessions.yaml.

  Core fields (canonical): session_name, status, last_seen, sandbox.
  Extra fields (autobahn-owned): backend, launched_at, harness, pid.
  """
  model_config = ConfigDict(extra="ignore")

  # Canonical fields
  session_name: str | None = None
  status: SessionStatus = SessionStatus.ABSENT
  last_seen: datetime | None = None
  sandbox: str | None = None

  # autobahn-extra (written by autobahn, ignored by spec-driver)
  backend: str | None = None
  launched_at: datetime | None = None
  harness: str | None = None
  pid: int | None = None


class SessionsFile(BaseModel):
  model_config = ConfigDict(extra="ignore")

  artifact: ArtifactBlock
  sessions: dict[str, SessionEntry] = {}
```

**Field mapping:**

| Old | New | Notes |
|-----|-----|-------|
| `session_id` | `session_name` | Renamed; backend-level identifier |
| `role` | dict key | Role enum value as string |
| `status` | `status` | Unchanged |
| `last_activity` | `last_seen` | Renamed to match canonical |
| `backend` | `backend` | Kept as autobahn-extra |
| `launched_at` | `launched_at` | Kept as autobahn-extra |
| `harness` | `harness` | Kept as autobahn-extra |
| `pid` | `pid` | Kept as autobahn-extra |

Dict semantics: one entry per role. A second spawn for the same role overwrites the previous entry. This is correct — one active session per role per artifact.

Canonical example allows `session_name: null` with `status: absent` — this represents a role slot with no active session.

### 4.3 Writer update

```python
# Canonical fields that must always be present (even when None)
_CANONICAL_SESSION_FIELDS = {"session_name", "status", "last_seen"}

def _serialize_session_entry(entry: SessionEntry) -> dict:
  """Serialize a SessionEntry, preserving canonical nulls."""
  data = entry.model_dump(mode="json", exclude_none=True)
  # Canonical required fields must be present even when None
  for field in _CANONICAL_SESSION_FIELDS:
    if field not in data:
      data[field] = None
  return data

def write_sessions_file(path: Path, sessions: SessionsFile) -> None:
  data = {
    "schema": _SESSIONS_SCHEMA,
    "version": write_version(_SESSIONS_SCHEMA),
    "artifact": sessions.artifact.model_dump(mode="json"),
    "sessions": {
      role: _serialize_session_entry(entry)
      for role, entry in sessions.sessions.items()
    },
  }
  tmp = path.with_suffix(".tmp")
  tmp.write_text(yaml.dump(data, default_flow_style=False, sort_keys=False))
  tmp.rename(path)
```

`exclude_none=True` keeps autobahn-extra fields clean, but canonical required fields (`session_name`, `status`, `last_seen`) are always emitted — even as `null` — because spec-driver's `MetadataValidator` requires their presence. The `status` field has a default (`absent`) so it's never None in practice, but the pattern is defensive.

**Maintainability note:** `_CANONICAL_SESSION_FIELDS` must stay in sync with the model. Add a contract test: `assert _CANONICAL_SESSION_FIELDS <= set(SessionEntry.model_fields)`. The `exclude_none=True` approach drops all None values including autobahn-extra; if a future extra field needs explicit null semantics, refactor to field-level exclusion.

### 4.4 Spawn persistence update

```python
def _persist_new_session(
  context: WorkflowContext,
  handle: SessionHandle,
  harness_name: str,
  result: OperationResult[SessionHandle],
) -> None:
  sessions_path = context.artifact_dir / "sessions.yaml"
  try:
    if context.sessions is not None:
      sf = context.sessions
    else:
      sf = SessionsFile(
        artifact=ArtifactBlock(
          id=context.artifact_id,
          kind=context.artifact_kind,
        ),
      )
    role_key = handle.role.value
    existing = sf.sessions.get(role_key)
    if existing is not None and existing.status == SessionStatus.ACTIVE:
      logger.warning(
        "Overwriting active session for role %s (session_name=%s)",
        role_key, existing.session_name,
      )
    entry = SessionEntry(
      session_name=handle.session_id,
      status=SessionStatus.ACTIVE,
      last_seen=handle.launched_at,
      backend=harness_name,
      launched_at=handle.launched_at,
    )
    sf = sf.model_copy(update={
      "sessions": {**sf.sessions, role_key: entry},
    })
    write_sessions_file(sessions_path, sf)
  except Exception:  # noqa: BLE001
    msg = f"Failed to persist session {handle.session_id} to sessions.yaml"
    logger.warning(msg, exc_info=True)
    result.warnings.append(msg)
```

### 4.5 persist_session_statuses update

```python
def persist_session_statuses(
  context: WorkflowContext,
  report: ReconciliationReport,
) -> None:
  # ... dead_ids collection unchanged ...

  changed = False
  new_sessions: dict[str, SessionEntry] = {}
  for role, entry in context.sessions.sessions.items():
    if entry.session_name in dead_ids and entry.status == SessionStatus.ACTIVE:
      new_sessions[role] = entry.model_copy(update={"status": SessionStatus.DEAD})
      changed = True
    else:
      new_sessions[role] = entry

  if changed:
    sf = context.sessions.model_copy(update={"sessions": new_sessions})
    write_sessions_file(sessions_path, sf)
```

Key change: matching on `entry.session_name` (was `entry.session_id`).

**Naming note:** `DriftItem.session_id` stores the session identifier from the drift detection perspective — it corresponds to `SessionEntry.session_name` and `SessionHandle.session_id`. The chain is: `SessionHandle.session_id` → `SessionEntry.session_name` → `DriftItem.session_id`. `DriftItem.session_id` is not renamed because it's a general-purpose field that identifies the drifted session regardless of source (handle, sessions.yaml entry, or future sources). The name divergence is intentional — `session_id` is the identifier concept, `session_name` is the canonical YAML field name.

## 7. Design Decisions

- **DEC-030**: Sessions.yaml conforms to canonical role-keyed dict schema. autobahn adds extra fields (`backend`, `launched_at`, `harness`, `pid`) that spec-driver's `MetadataValidator` tolerates (confirmed: validator does not reject unknown fields). Spec-driver never reads sessions.yaml; autobahn is sole reader and writer. **Session history is last-writer-wins:** a second spawn for the same role overwrites the previous entry. If the previous entry was still `active`, a warning is logged. No session history ledger in v1.
- **DEC-031**: `session_name` maps to `SessionHandle.session_id` — the backend-level identifier (e.g., `DE-099-reviewer`). Role is the dict key. The `{artifact_id}-{role}` construction convention in `Supervisor.spawn` is unchanged.
- **DEC-032**: Read-surface scope limited to fix breakage + fields DE-004 needs. Deferred: `domain_map`, `invariants`, `risk_areas`, `review_focus`, `known_decisions` on ReviewIndexFile; handoff `verification`, `git`, `design_tensions`, `unresolved_assumptions`, `decisions_to_preserve`; handoff `OpenItemBlock` restructure. `extra="ignore"` tolerates these fields.
- **DEC-033**: `FindingDisposition.authority` becomes `DispositionAuthority` enum and is **required** (was `str | None`). This is a constructor-level breaking change — any code that creates `FindingDisposition` without `authority` will fail pydantic validation. Gains `actor_id`, `backlog_ref`, `resolved_at`, `superseded_by`, `timestamp` — fields required for action-specific validation in DE-004 review operations.
