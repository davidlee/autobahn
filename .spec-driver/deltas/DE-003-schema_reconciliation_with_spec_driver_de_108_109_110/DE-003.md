## 1. Summary & Context

- **Predecessor**: DE-002 — internal hardening and session persistence
- **Implementation Plan**: [IP-003](./IP-003.md)
- **Design Revision**: [DR-003](./DR-003.md)
- **Change Drivers**: spec-driver DE-108 (structured CLI), DE-109 (review state machine), DE-110 (schema export) all landed — autobahn's models are now stale

## 2. Motivation

- spec-driver DE-109 removed `warming` from `BootstrapStatus` and `blocked` from `ReviewStatus` — autobahn's contract tests should fail
- spec-driver DE-110 exports canonical schemas; the `workflow.sessions` schema uses a role-keyed dict, not autobahn's current list model
- spec-driver DE-109 added `judgment_status` to review-index and enriched the review-findings v2 model with structured dispositions
- Handoff schema gained `key_files`, `verification`, structured `open_items`
- Review-index schema gained `judgment_status`, `domain_map`, `invariants`, `risk_areas`, `review_focus`, `known_decisions`
- Without reconciliation, autobahn cannot correctly load workflow directories produced by current spec-driver, and cannot build review operations (DE-004)

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Enum reconciliation: remove `warming` from `BootstrapStatus`, `blocked` from `ReviewStatus`
  - Sessions.yaml model realignment to canonical role-keyed dict schema
  - Sessions.yaml writer updated to produce canonical format
  - Review-index model updated with new fields (judgment_status, domain_map, etc.)
  - Review-findings model updated with disposition model, reviewer_role, round session metadata
  - Handoff model updated with new fields (key_files, verification, structured open_items)
  - SCHEMA_VERSIONS updated if needed
  - All contract tests updated to match
  - All existing tests pass or updated for new model shapes
- **Operational Constraints**: No new API functions — model + contract layer only
- **Dependencies**: None — all upstream deltas are landed

## 4. Out of Scope

- Review operations API (DE-004)
- CLI wrapper / tool invocation adapter for spec-driver
- New test fixtures beyond what's needed for model validation
- Round session metadata schema definition (autobahn owns this; define when review ops land)

## 5. Approach Overview

- **Key Changes**:
  - `models/enums.py` — remove `warming`, `blocked`
  - `models/artifacts.py` — update `SessionsFile`, `SessionEntry`, `ReviewIndexFile`, `ReviewFindingsFile`, `HandoffFile` and sub-models
  - `artifacts/schema.py` — update SCHEMA_VERSIONS if schema versions changed
  - `artifacts/writer.py` — update `write_sessions_file` for dict-based sessions
  - `artifacts/loader.py` — no structural changes expected (pydantic models handle shape)
  - `api/functions.py` — update `_persist_new_session` and `persist_session_statuses` for new sessions model
  - `runtime/reconcile.py` — update orphan detection for dict-based sessions
  - `tests/` — update contract tests, fixtures, loader tests, writer tests, API tests, reconcile tests

## 6. Verification Strategy

- **Acceptance Criteria**:
  - `BootstrapStatus` enum values match spec-driver canonical values (no `warming`)
  - `ReviewStatus` enum values match spec-driver canonical values (no `blocked`)
  - Sessions.yaml round-trip test passes with role-keyed dict format
  - All models load canonical schema examples without error
  - `just check` passes
  - Contract tests verify against spec-driver canonical enum values

## 7. Risks & Mitigations

- **Risk**: Sessions.yaml model change is breaking for any existing sessions.yaml files — _Likelihood_: low (no production use yet) — _Impact_: low — _Mitigation_: v1 only, no migration needed
- **Risk**: Enum removal may break downstream code that references removed values — _Likelihood_: low — _Impact_: low — _Mitigation_: grep for `warming` and `blocked` usage before removing
